Why HooCode?

You stay in control — always.

Most AI editors make changes silently. HooCode gates every file edit and shell command behind your approval, so nothing lands in your codebase without your knowledge.

Feature HooCode Typical AI editor
Edit approval ✓Prompted before every file change Changes land silently
Shell gating ✓Shell commands need explicit sign-off Runs commands autonomously
Mode-driven focus ✓Ask / Plan / Build / Debug modes Single chat interface for everything
LLM provider ✓25+ providers, swappable in one line Locked to one vendor
Extensibility ✓MCP servers, TypeScript extensions, profiles Closed plugin system
Binary distribution ✓Self-contained binary, no Node.js needed IDE or cloud subscription required

Workflow

Four focused modes, one agent.

Switch modes at any time with /mode <name>. Each mode configures a different system prompt and permission set so the agent stays on-task.

ask

Ask

Read-only Q&A about your codebase. The agent never writes a file or runs a command — perfect for exploration and understanding.

plan

Plan

The agent explores your repository and writes .hoocode/plan.md. You review and approve before a single file changes.

build

Build

Implements the approved plan. Every edit and shell command is gated — you confirm each step or allow-list safe operations.

debug

Debug

Root-cause analysis mode. The agent explains what's wrong and how to fix it, but does not modify any files.

Packages

Four packages, one coherent stack.

Use the CLI out of the box, or compose the individual packages into your own tools.

🤖

hoocode-agent

The interactive CLI (hoocode / hoo). Multi-mode workflow, permission system, plan approval, MCP integration, and a self-contained binary build.

⚙️

hoocode-agent-core

Stateful agent runtime with event streaming, parallel / sequential tool execution, beforeToolCall hooks, and abort / steer support. Build your own agents on top.

🌐

hoocode-ai

Unified streaming API for 25+ LLM providers — tool calling, vision, reasoning, image generation, cost tracking, and cross-provider context hand-offs.

🖥️

hoocode-tui

Differential terminal UI with ANSI-aware rendering, overlay system, markdown + syntax highlighting, inline images, and IME / CJK support.

Get Started

Up and running in 30 seconds.

Install globally via npm, then run hoocode inside any project.

$ npm install -g @kolisachint/hoocode-agent
$ export ANTHROPIC_API_KEY=sk-ant-… # or OPENAI_API_KEY, GEMINI_API_KEY, …
$ cd my-project
$ hoocode
# Switch mode at any time
> /mode plan
> Add a /healthz endpoint to the Express server

# Review .hoocode/plan.md, then execute it
> /approve
